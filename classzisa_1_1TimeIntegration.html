Evolve solution from t to t + dt.

The caller must ensure that the time-step is sufficiently small to ensure that the integration is stable.

Note: the update will not modify u0. Therefore, the returned smart pointer does not point to the same object as u0.

Note: RungeKutta takes shared ownership of u0 and therefore, can modify u0 in a later call.

These two comments imply that code such as 

auto u1 = rk.compute_step(u0, t, dt);
auto du = difference(*u0, *u1);

is safe.

WARNING: The two comments also imply that passing the same buffer twice is not safe.
